# fraud_model_1
This project aims to build a fraud detection model and optimize decision thresholds to balance fraud prevention and customer experience.
Note: The model performs well in identifying fraud patterns, but threshold tuning is critical to reduce false positives.
A hybrid strategy (review + blocking) is recommended for real-world deployment.

Tools:
Data exploration
Handling imbalanced data;

Logistic regression model;

Threshold tuning;

Findings:
High recall (~83%);

Precision improved from 0.07 to 0.40;

Significant reduction in false positives.
